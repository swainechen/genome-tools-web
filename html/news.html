Mostly minor bug fixes. Please update. As always, files are at: http://sourceforge.net/project/showfiles.php?group_id=45954 Main fix is to change to new "head" command syntax. November 29, 2007

The server has been rebuilt, and all redirects from sourceforge.net are now back to http://genome-tools.stanford.edu. The backup http://hultgren.wustl.edu/genome-tools will still be available for a while, and should have the same functionality and genome data.

New org-map.standard and org-map.directories files are available. Run 'update-genome.pl -all' to update to the new files and download the new genomes. There are now 566 microbial genomes total, with 1000 chromosomes/plasmids.

This will likely be the last time I update these files in their current form. The number and rate of new genome sequences is too much to keep up with. Some fields that don't really have that much utility may be deprecated, and orgcode syntax may be expanded to allow for longer orgcodes. This will of course require an upgrade to the genome-tools scripts and modules. Stay tuned.

December 27, 2004

Genome-tools 1.2 released.
It is highly recommended that all installations upgrade to this new release. There is a new upgrade script to simplify the upgrade procedure, which should work if there is a previously installed genome-tools.conf file. This release fixessome parsing errors and processing errors which resulted in possibly missing some intergenic sequence (although in some cases we can't do anything about NCBI .ptt files specifying illogical situations) and possibly not preprocessing some genome data files.

Please let me know if you find any errors or have any problems. Look for an updated org-map.standard file in the next week or two (i.e. run update-genome.pl -all).

Major fixes/changes:
- Added upgrade-genome-tools.pl script.
- Some Genbank file formats have changed slightly since the initialed genome-tools release. Parsing errors should now be gone.
- Downloading and preprocessing genomes when they were already installed or when multiple sets of files were in the same directory sometimes did not update the .ptt file. This has been fixed.

April 10, 2004

New web server is up at http://genome-tools.stanford.edu. Please update any bookmarks. All genome files are now stored uncompressed, as space limitations are much relaxed, so things should run a bit quicker in general.

Also, two new graphical tools are available and highlighted in gray. The first allows drawing of circular maps, marking off certain positions (for example, copy the output from the "Position" function and paste it into the input for the "Circular-Chromosome location" function). The second allows drawing a linear map of a specified region of the chromosome, providing a to-scale view of the genes in that regions of the chromosome.

December 7, 2003

Genome-tools version 1.1 is released now. Download at http://prdownloads.sourceforge.net/genome-tools/genome-tools-1.1.tar.gz?download.

Major changes concern installation and maintenance.

Briefly, there is an installation script and a new configuration file scheme which allows there to be a system-wide as well as user-specific configuration file. The setup script helps write the system-wide configuration file and installs all the files.

There is a new script, update-genome.pl, which vastly simplifies the management of genome data files in terms of installing them for use locally. There is a standard org-map file on the web site which this script looks at, and it will download the data files and set up the org-map file automatically if run in batch mode.

Also, genome files for Sulfolobus species have been uncompressed on the web site. Searches on these genome files from the web interface should be faster now.

July 29, 2003

Due to space constraints on the sourceforge.net server, many of the genome files are compressed on this web server. This causes some slowdown as they need to be uncompressed every time a request is made. For genomes that are more frequently accessed, we have stored the data files in uncompressed form, which should improve the performance of searching through the genome sequence. Currently, this applies to the Caulobacter crescentus, Escherichia coli K12, Mycobacterium tuberculosis CDC1551, and Salmonella typhimurium LT2 genomes.

Also, we've been working on installation and automation scripts so that installing and maintaining genome-tools locally is much easier. If you have a local installation of genome tools, check back here in the near future for further updates.

And if you are interested in installing genome-tools locally to take advantage of automated batch processing across many genomes, but have been stumped by the difficult installation, please also check back here soon for the new release we're preparing!

June 2, 2003

Getting genome-tools set up locally is admittedly still pretty hairy. We are working on this.

The first thing we've done is move towards using the curated genome files from GenBank. As these files are generally more consistent, we will move away from using sanemaker files to "clean up" annotation files.

There is now a program in the CVS tree called update-genome.pl which will help in downloading the correct files from GenBank (including the rnatab files!), setting up the org-map line, and preprocessing the genome files. All you need to know is the location of your org-map file and the name of the directory on the GenBank FTP server where the files are stored. The list of directories as of June 2, 2003 can be found here.

The script is still not perfect yet - it ought to be able to autodetect where your org-map file is, it should be able to handle a pre-existing org-map line if you're downloading the same genome, etc. etc. Right now it works ok for building the initial org-map file, though. When we get it more polished we will be making another release of the entire package.

Note that if you're going to download the update-genome.pl script, it can be found in the sbin directory under the CVS tree. You will also need new versions of mergeptt.pl, preproc-new-genome.pl, and rnatab-to-ptt.pl (all in the sbin directory) as well as a new version of Orgmap.pm (in the lib directory).

Hope this helps. Let us know if there are questions/comments/suggestions. Thanks.

March 26, 2003

To help with downloading and installing genomes for use with genome-tools, we have made all the files we use for preprocessing available. There are two files, sanemakers-031703.tar.gz and rnatab-031703.tar.gz which contain the sanemakers and rnatab files we use on our machine. A list of genomes we have successfully used with genome-tools can be found here.

Most of the data files for these genomes were taken from the original GenBank submissions, and they can be downloaded at ftp://ftp.ncbi.nih.gov/genbank/genomes/Bacteria. Note, the data files at ftp://ftp.ncbi.nih.gov/genomes/Bacteria are not the same, they are the curated versions. These are less stable than the orgiinal submissions, so we do not have files for those. Many of them may not need a sanemaker.pl file, however. Make sure that the data files have the same name before you use the sanemaker.pl files. (For example, the original GenBank submission for Caulobacter crescentus has data files starting with AE005673. The curated version has data files starting with NC_002696.)

A note about preprocessing - the basic procedure is to download all the data files from the GenBank site into a separate directory. Additional chromosomes or plasmids must all be in separate directories (we keep them in subdirectories). Then, if there is an rnatab file available (see below), place it in the same directory. If there is a sanemaker.pl file available (see below), place it in the same directory. Then, add the appropriate line to the org-map file (our org-map file is given as an example in org-map.sample) in the lib directory. Then, if you're running bash, all you need to do is run:

bash-prompt# preproc-new-genome.pl org-code 2>&1 > preproc-new-genome.log

Or, if you're using csh,

csh-prompt# preproc-new-genome.pl org-code >& preproc-new-genome.log

within the new directory containing all the genome files. Then you should be ready to go. If you don't have tRNAScan or the GenBank BLAST software installed, don't worry, you can ignore the errors it gives you on those parts of the preprocessing.

Finally, if you write a new sanemaker.pl file for a new genome or have changes you think should be made to existing files, please let us know by email. Also let us know if you have problems/difficulties with these instructions. We are working on a simpler system for handling all these data files and different versions, but hopefully these files will get you started.

March 3, 2003

A 1.0.1 release is now available at http://prdownloads.sourceforge.net/genome-tools/genome-tools-1.0.1.tar.gz?download. This (finally) has some installation instructions. You still need to know your way around your system a little bit and be familiar with some command line utilities in order to get this to work, though. Our apologies to those who may be having trouble installing the genome-tools package. Hopefully this will help.

Still missing, however, are more explicit instructions on how to get genome files ready for use with genome-tools. We've got close to 80 genomes processed and ready to go on our machine, and we'll try to make available all the sanemaker.pl files we have soon -- the rest of the files you will need are available on GenBank. The major delay is that we need to make sure that the right version of the genome data and annotation files are used with each of our post-processing script, and we're working on making that a bit simpler, as well as simplifying the downloading of genomes from GenBank.

If you have questions, problems, requests, or comments, we'd love to hear from you. You can get our email addresses from the project home page, at http://sourceforge.net/projects/genome-tools.
